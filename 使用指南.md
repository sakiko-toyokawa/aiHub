## 快速开始

### 1. 环境要求

- **Docker** (推荐) 或 **Python 3.11+** + **Node.js 20+**
- **PostgreSQL** (如果使用 Docker 则自动包含)
- 稳定的网络连接

### 2. 下载项目

```bash
git clone <项目地址>
cd ai-knowledge-hub
```

### 3. 配置环境变量

复制示例配置文件：

```bash
cp .env.example .env
```

编辑 `.env` 文件，填入你的 API Key 和账号信息：

```env
# AI API Key (至少填一个)
OPENAI_API_KEY=sk-your-openai-key
DEEPSEEK_API_KEY=sk-your-deepseek-key

# 平台账号 (用于抓取内容)
ZHIHU_COOKIE=z_c0=你的知乎cookie
BILIBILI_SESSDATA=你的bilibili-sessdata
TWITTER_USERNAME=你的推特用户名
TWITTER_PASSWORD=你的推特密码

# 邮件配置 (用于每日推送)
SMTP_HOST=smtp.qq.com
SMTP_USER=your@qq.com
SMTP_PASSWORD=你的授权码
EMAIL_TO=接收邮箱@example.com
```

### 4. 启动服务

#### 方式一：Docker (推荐)

```bash
# 一键启动所有服务
docker-compose up -d

# 查看日志
docker-compose logs -f

# 停止服务
docker-compose down
```

#### 方式二：手动启动

**后端：**

```bash
cd backend

# 创建虚拟环境 (推荐)
python -m venv venv
source venv/bin/activate  # Linux/Mac
# 或: venv\Scripts\activate  # Windows

# 安装依赖
pip install -r requirements.txt

# 初始化数据库
python -c "from app.database import Base, engine; Base.metadata.create_all(bind=engine)"

# 启动服务
uvicorn app.main:app --reload --port 8000
```

**前端：**

```bash
cd frontend

# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

### 5. 访问应用

- **前端界面**: http://localhost:3000
- **API 文档**: http://localhost:8000/docs

## 详细配置

### 获取账号凭证

#### 知乎 Cookie

1. 用浏览器登录知乎
2. 按 F12 打开开发者工具 → Application/应用 → Cookies
3. 找到 `z_c0` 字段，复制其值
4. 填入 `.env` 文件: `ZHIHU_COOKIE=z_c0=复制的值`

**注意**: Cookie 有过期时间，如抓取失败请重新获取

#### Bilibili SESSDATA

1. 用浏览器登录 Bilibili
2. 按 F12 打开开发者工具 → Application/应用 → Cookies
3. 找到 `SESSDATA` 字段，复制其值
4. 填入 `.env` 文件: `BILIBILI_SESSDATA=复制的值`

#### Twitter/X 账号

直接在 `.env` 中填入：

```env
TWITTER_USERNAME=你的用户名
TWITTER_PASSWORD=你的密码
TWITTER_EMAIL=你的邮箱(可选)
```

Twitter 使用 `twikit` 库，无需申请 API Key

#### GitHub Token (可选)

GitHub 抓取公开仓库无需 Token，但加 Token 可提高限额：

1. 访问 https://github.com/settings/tokens
2. 创建 Token (不需要任何权限)
3. 填入 `.env`: `GITHUB_TOKEN=你的token`

### AI 服务商选择

项目支持多个 AI 服务商，在 `.env` 中设置默认：

```env
DEFAULT_AI_PROVIDER=openai  # 可选: openai, claude, deepseek, glm, kimi, minimax
```

**各服务商特点：**

| 服务商 | 优势 | 注册地址 |
|--------|------|---------|
| OpenAI | 质量高，速度快 | https://platform.openai.com |
| Claude | 长文本能力强 | https://anthropic.com |
| DeepSeek | 中文好，价格低 | https://platform.deepseek.com |
| GLM | 国内模型 | https://open.bigmodel.cn |
| Kimi | 长文本支持 | https://platform.moonshot.cn |
| MiniMax | 国内模型 | https://platform.minimaxi.com |

### 定时任务配置

默认配置：

```env
# 每2小时抓取一次
CRAWL_SCHEDULE=0 */2 * * *

# 每3小时总结一次新内容
SUMMARIZE_SCHEDULE=0 */3 * * *

# 每晚10点发送邮件
EMAIL_SCHEDULE=0 22 * * *
```

修改后需要重启服务生效。

## 日常使用

### 界面操作

#### 首页 (Dashboard)

1. **筛选内容**: 点击顶部标签筛选不同平台
2. **搜索**: 在搜索框输入关键词
3. **收藏**: 点击卡片上的收藏按钮
4. **查看详情**: 点击卡片进入详情页
5. **阅读原文**: 点击原文链接跳转

#### 详情页

- 查看 AI 生成的完整总结
- 查看关键要点
- 查看原始内容
- 标记已读/收藏

#### 设置页

- 配置 AI 服务商和 API Key
- 管理数据源 (启用/禁用平台)
- 设置邮件通知
- 查看系统状态

### API 接口

**获取总结列表:**

```bash
curl "http://localhost:8000/api/summaries?page=1&page_size=20"
```

**按平台筛选:**

```bash
curl "http://localhost:8000/api/summaries?platform=github"
```

**搜索:**

```bash
curl "http://localhost:8000/api/summaries?search=GPT"
```

**标记已读:**

```bash
curl -X POST "http://localhost:8000/api/summaries/1/read"
```

**收藏:**

```bash
curl -X POST "http://localhost:8000/api/summaries/1/favorite"
```

完整 API 文档: http://localhost:8000/docs

## 常见问题

### Q: 爬虫无法获取内容？

**检查清单:**

1. 检查网络连接
2. 检查账号凭证是否过期 (Cookie/SESSDATA)
3. 查看后端日志: `docker-compose logs backend`
4. 确认平台没有被封 IP

**知乎 Cookie 过期:**

```bash
# 重新登录知乎，获取新的 z_c0
# 更新 .env 文件
# 重启服务
docker-compose restart backend
```

### Q: AI 总结失败？

1. 检查 API Key 是否正确
2. 检查 API Key 是否有余额
3. 尝试切换其他 AI 服务商
4. 查看日志中的错误信息

### Q: 邮件发送失败？

**QQ 邮箱配置示例:**

```env
SMTP_HOST=smtp.qq.com
SMTP_PORT=587
SMTP_USER=123456@qq.com
SMTP_PASSWORD=授权码(不是登录密码)
```

获取授权码: QQ邮箱 → 设置 → 账户 → 开启 SMTP → 获取授权码

### Q: 如何手动触发抓取？

```bash
# 进入后端容器
docker-compose exec backend bash

# 运行爬虫测试
python -c "
import asyncio
from crawler import get_crawler
crawler = get_crawler('github')
results = asyncio.run(crawler.fetch())
print(f'抓取到 {len(results)} 条内容')
"
```

### Q: 数据存储在哪里？

- **使用 Docker**: 数据存储在 `postgres_data` Docker Volume
- **手动部署**: 在配置的 PostgreSQL 数据库中

备份数据:

```bash
docker-compose exec postgres pg_dump -U user ai_knowledge > backup.sql
```
